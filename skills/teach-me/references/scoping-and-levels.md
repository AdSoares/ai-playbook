# Reference — Scoping, Time Budget & Levels

> Load this when scoping a subject, deciding whether to slice it, defining the 5 levels, budgeting
> time, or diagnosing prerequisites — i.e. everything before generating the capsule.

## 1. Classify the subject first

| Type | Example | Action |
|---|---|---|
| Adequate | "Intro to Docker" | Proceed normally. |
| Too broad | "Learn programming" | Suggest slices. |
| Too abstract | "Understand economics" | Propose sub-topics. |
| Needs long practice | "Play violin well" | Generate a starter capsule; warn mastery needs training. |
| High risk | health, legal, financial, safety | Strong caveats + trustworthy sources (see sources-and-safety). |
| Fast-changing | tools, laws, APIs, trends | Use recent sources; record access dates. |
| Non-public / protected | paid, proprietary, confidential | Don't copy; only guide or use permitted sources. |

### Heuristics — suggest a slice when
- The subject has many independent subdomains.
- The core content needs more than the budget.
- The chosen level is advanced/expert without a prior base.
- The topic depends on long practice.
- The topic has many controversies or schools of thought.
- The learner's goal is too generic.

### Overflow rule
If the subject needs more than the budget to be treated properly: (1) say it's too broad,
(2) suggest slices, (3) propose alternative focused capsules, (4) explain what will be left out,
(5) let the learner pick a narrower capsule.

> Example: "Artificial Intelligence" is too broad for ≤10h → slice into: GenAI for everyday users
> (introductory) · ML fundamentals (beginner) · LLMs for developers (intermediate) · RAG in
> practice (advanced) · AI product strategy (expert).

## 2. Time budget (default ≤10h, never exceed)

| Block | Suggested time |
|---|---:|
| Diagnosis & goal setting | 20–40 min |
| Prerequisite review | 30–90 min |
| Main lessons | 2–4 h |
| Recommended external content | 1–3 h |
| Examples & case studies | 1–2 h |
| Exercises & active recall | 1–2 h |
| Final assessment | 30–90 min |
| Review & next steps | 20–40 min |

The total must never exceed the budget. Trim *Útil/Useful* topics first (see capsule-blueprint).

## 3. Always present 5 levels

| Level | Description | Expected outcome |
|---|---|---|
| Introductory | Overview & vocabulary | Knows what it is, what it's for, the main ideas. |
| Beginner | Fundamentals & first examples | Understands core concepts, recognizes simple cases. |
| Intermediate | Application to common problems | Applies knowledge to typical situations. |
| Advanced | Trade-offs, exceptions, hard problems | Analyzes complex scenarios, avoids sophisticated errors. |
| Expert | Decision, critique, design, teaching | Evaluates approaches, teaches, designs solutions, knows the limits. |

The **expert** level inside a time-boxed capsule = studying from an expert's perspective (criteria,
limitations, decisions, risks, debates, advanced applications), NOT becoming a professional expert.

## 4. Prerequisite diagnosis (before building)

Output the prerequisites, classified **Required / Recommended / Useful**:

```
## Prerequisites to study {subject} at the {level} level
1. Prerequisite A — Required
2. Prerequisite B — Recommended
3. Prerequisite C — Useful
```

Then run a quick diagnostic (a few targeted questions), and route on the result:

| Result | Action |
|---|---|
| Prerequisites mastered | Proceed to the main capsule. |
| Light gaps | Add a quick review at the start of the capsule. |
| Medium gaps | Suggest a preparatory mini-capsule. |
| Severe gaps | Strongly recommend another subject/level first. |
| Learner insists | Proceed, but record a caveat. |
