# Reference — Sources & Safety

> Load this when curating sources, handling external or foreign-language content, or treating
> sensitive/high-risk subjects. The anti-hallucination and copyright rules here are non-negotiable.

## 1. Source types & permitted use

| Type | Permitted use |
|---|---|
| Public domain | Freer use, respecting context and attribution. |
| Open license / OER | Per the license. |
| Official documentation | Reference and summarize. |
| Academic papers | Summarize and cite. |
| Free videos | Reference, with estimated time. |
| Protected content | Don't copy in full; cite, summarize, or recommend. |
| Paid content | Don't depend on it as a required source unless the learner asks. |

## 2. Selection criteria (priority order)
1. Official sources → 2. Academic → 3. Institutional → 4. Recognized authors →
5. Up-to-date material → 6. Matches the learner's language → 7. Good didactics →
8. Fits the time budget.

## 3. Source record (`sources.html`)
Record **every** source with:
```yaml
title:
author_or_institution:
type:
link:
language:
license:
estimated_time:
access_date:
why_used:
related_topics_or_excerpts:
```

## 4. Foreign-language sources
May be used when better or more authoritative, but: warn the learner; prefer the learner's language
at equal quality; summarize/explain in the learner's language; record the source language.

## 5. Recommended external content
May recommend texts/videos/audio, but the time counts inside the budget. The capsule must never
depend exclusively on external content — always provide a sufficient own explanation.
```
### Required external content
1. Video: "Intro to X" — 18 min — gives a simple overview — Link: …
2. Article: "Official guide to Y" — ~25 min — official source — Link: …
```

## 6. Risks & mitigations

**Exaggerated promise** — never "you'll be an expert in any subject in 10h". Prefer "a structured
capsule for the best possible mastery of this subject and level within the budget".

**Hallucination** — record sources; prefer official sources; separate fact / interpretation /
recommendation; include `caveats.html`; use access dates; flag uncertainty.

**Protected content** — don't copy protected content in full; short quotes only when necessary;
prefer open sources; record licenses; link external content.

**Sensitive subjects** (health, legal, financial, safety, politics, regulation): include caveats;
use trustworthy sources; do not replace a professional; state that rules can change; recommend
specialist consultation when relevant.
