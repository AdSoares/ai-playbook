Caveats

Scope & expectations

This is a structured introduction, not professional mastery. It makes you productive with Docker locally; it does not make you a production/ops practitioner (see the ignorance map).

Things that change over time

  • The Docker CLI evolves. Commands here are stable, but flags and defaults can shift — always cross-check the official docs (dated 2026-06-13 in sources).
  • Base image tags (python:3.11-slim, postgres:16) age; newer versions appear. Pin deliberately.
  • Docker Desktop licensing has terms that depend on company size — verify before commercial use.

Platform notes

  • Containers are a Linux-kernel technology. On Windows/macOS, Docker runs a lightweight Linux VM under the hood — behavior is consistent but not literally "no VM" on those hosts.
  • File paths, line endings, and volume permissions can differ across host OSes.

Safety

Default isolation is not a hardened security boundary. Don't run untrusted images casually, don't bake secrets into images, and review official security guidance before exposing anything beyond localhost.
