5-minute quick review (if you had light gaps)

Process: a running program. The same program file can run as many processes.

Filesystem: the tree of files a program sees. Two programs can see different trees.

Port: a numbered "door" on a machine where a network service listens. A web server might listen on port 80; you reach it at http://localhost:80.

Package vs. running program: an installer/image is a frozen artifact on disk; the running program is what exists in memory after you start it. Hold this distinction — it is exactly the image vs. container idea you'll meet in Lesson 2.
