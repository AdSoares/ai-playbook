Exercises

Closed-book where possible. Try before peeking at the hint.

Hands-on

  1. Run hello-world. What does the output confirm about your install?
  2. Run nginx detached on host port 8081, open it, then stop and remove it.
  3. Run Postgres 16 with a named volume. Stop and remove the container, run it again with the same volume — your data (if you'd created any) is still there. Why?
  4. Build the Lesson 5 app as hello-web, then rebuild as hello-web:v2 with different text and run both at once on ports 8080 and 9090.

Conceptual

  1. Explain image vs. container to someone using an analogy that is NOT "class/object".
  2. Reorder this Dockerfile to maximize cache reuse and say why: COPY . . then RUN pip install -r requirements.txt.
  3. A teammate says "EXPOSE 80 should make my site load but it doesn't." Diagnose it.
Hints. #3: volumes live outside the container's writable layer. #6: copy requirements.txt and install before COPY . .. #7: EXPOSE documents, -p publishes.
