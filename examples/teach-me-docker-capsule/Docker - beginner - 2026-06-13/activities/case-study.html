Case study — "Run our tool locally"

Apply the whole capsule to a realistic scenario.

Scenario. A teammate shares a small internal web tool as a folder with app.py and a Dockerfile. They also mention it needs a Postgres database and a config value APP_ENV=local.

Your tasks

  1. Start a Postgres 16 container with a named volume and a password. Write the command.
  2. Build the tool's image as internal-tool. Write the command.
  3. Run the tool, published on host port 8080, passing APP_ENV=local. Write the command.
  4. The tool exits immediately. Which command do you run first to find out why?
  5. State one thing that would persist after you remove both containers, and one thing that would not.

Reference solution (peek after trying)

1) docker run -d -p 5432:5432 -e POSTGRES_PASSWORD=secret -v pgdata:/var/lib/postgresql/data postgres:16 2) docker build -t internal-tool . 3) docker run -d -p 8080:80 -e APP_ENV=local internal-tool 4) docker logs <id> 5) Persists: the pgdata volume. Lost: anything written inside the tool container's filesystem (no volume).
