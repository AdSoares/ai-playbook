teach-me Docker · Objectives

Objectives & applications

This is the learning contract. The final assessment checks each item.

By the end of this capsule you will be able to:

  1. Explain what a container is and the problem it solves (in your own words).
  2. Distinguish an image from a container, and a registry from a local cache.
  3. Run any public image with one command, including publishing a port.
  4. Write a basic Dockerfile and build an image from it.
  5. Persist data with a volume and know why container filesystems are ephemeral.
  6. Avoid the most common beginner mistakes (latest tag, lost data, huge images).
  7. Know when to go deeper (Compose, networking, production) and when Docker is not the answer.

Where this is useful

  • Running a database or tool locally without polluting your machine.
  • Reproducing "works on my machine" reliably across teammates and CI.
  • Packaging a small app so anyone can run it with one command.

When this knowledge is NOT enough

Production deployment, scaling, secrets management, and multi-service orchestration are out of scope here — see the ignorance map and deepening.
