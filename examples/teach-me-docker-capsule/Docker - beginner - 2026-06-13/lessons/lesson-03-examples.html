Lesson 3 — Run, build, ports & volumes (hands-on)

Estimated time: 1h00 · Bloom focus: Apply

Requires Docker installed (Docker Desktop or Engine). If you can't install now, read along — every command has its expected effect described.

1. What you'll learn

The everyday commands: run a container, see what's running, publish a port, persist data.

2. Why it matters

This is 80% of daily Docker use. Internalize these six commands and you're productive.

3. Simple explanation — the core commands

docker run hello-world # pull + run a tiny test image docker run -d -p 8080:80 nginx # run nginx in background, map host 8080 -> container 80 docker ps # list running containers docker stop <id> # stop one docker rm <id> # remove it docker images # list local images

After the nginx command, open http://localhost:8080 — you'll see the nginx page.

4. Concrete example — ports

-p 8080:80 means "send traffic arriving on my machine's port 8080 to the container's port 80". The number before the colon is host; after is container. Forget -p and the service runs but you can't reach it from your browser.

5. Correct technical view — volumes & ephemerality

A container's writable layer dies with the container. To keep data, mount a volume:

docker run -d -p 5432:5432 \ -e POSTGRES_PASSWORD=secret \ -v pgdata:/var/lib/postgresql/data \ postgres:16

The -v pgdata:/var/lib/postgresql/data stores the database files in a named volume pgdata that survives docker rm. -e sets an environment variable inside the container.

6. Common mistakes

  • Swapping the port order (-p 80:8080) and then "the site won't load".
  • Storing important data in the container with no volume, then losing it on rm.

7. Practical case

You need Postgres for a local project. Instead of installing it, you run the command above. Delete the container when done; the data stays in pgdata for next time.

8. Recall questions

In -p 3000:80, which port do you open in the browser?

localhost:3000 (the host port, before the colon). localhost:80 (the container port, after the colon).

Why might data "disappear" after you remove a container?

The writable layer is discarded on removal unless a volume was mounted. Docker compresses old containers automatically.

What does -d do in docker run -d ...?

Runs the container detached (in the background). Deletes the container after it starts.

9. Mini-exercise

Run nginx on host port 9000, confirm it in the browser, then stop and remove it. Write down the three commands you used.

10. Summary

run starts a container; -d detaches it; -p host:container publishes a port; -v name:/path persists data in a volume; -e sets env vars. ps, stop, rm, images manage the rest. Container filesystems are ephemeral — use volumes for anything you want to keep.

11. Terms for the glossary

volume, port mapping, detached mode, environment variable, ephemeral filesystem.
