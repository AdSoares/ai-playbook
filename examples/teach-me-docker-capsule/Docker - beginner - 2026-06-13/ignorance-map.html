Ignorance map

Honest boundaries of this capsule — what you can and cannot yet claim.

✅ This capsule covers

  • The container concept and the image/container/registry/Dockerfile model.
  • Running, building, port-publishing, and persisting data locally.
  • The common beginner mistakes and how to debug a container.

➖ It does NOT cover

  • Docker Compose and multi-container apps (next capsule).
  • Networking beyond a single published port (custom networks, DNS).
  • Production: orchestration (Kubernetes/ECS), scaling, health checks, secrets management.
  • Security hardening, non-root users, image scanning, supply-chain concerns.
  • Multi-stage builds and serious image-size optimization.

⏳ Needs more practice (not just reading)

  • Writing Dockerfiles fluently for different stacks.
  • Debugging real container failures under time pressure.

🚫 Don't yet claim

That you can "deploy and operate containers in production". You can develop locally with containers — a real, useful skill, but a different one.
