Assessment

The multiple-choice items below give instant feedback. For the open questions, return to the chat and ask the teach-me skill to grade you — it reads your history and avoids repeating questions.

Multiple choice

1. What best describes the image↔container relationship?

Image = immutable template; container = running instance created from it. Container = the file on disk; image = the process in memory. They are identical and interchangeable.

2. In docker run -p 8080:80 nginx, you open which URL?

http://localhost:8080 http://localhost:80 http://nginx:8080

3. Why can container data disappear after docker rm?

The writable layer is discarded unless a volume was mounted. Docker deletes images older than a day. Removing a container also clears the registry.

4. Why prefer python:3.11-slim over :latest?

Pinned tags make builds reproducible; latest can change. slim images run with admin rights automatically. latest cannot be used in a Dockerfile.

5. Inside a container, a web server should bind to…

0.0.0.0 — so forwarded host traffic is accepted. 127.0.0.1 — for better performance.
